    Depuis github

    1. Création de votre fork

    Pour créer votre document, il vous suffit de vous rendre sur Github, via le lien https://github.com/Oria-Organization/oria-bot/ qui est notre repo. La création d'un compte est requis, voir (Documentation github).
    Par la suite vous pourrez cliquer sur la flèche de "fork" (voir image 1), et ensuite "Create a new fork" (voir image 2).


    Vous vous retrouverez sur la page de l'image 1, il vous suffit donc de faire "Create fork".

    Tout le contenu du repo sont sous la possession d'Oria Production, la modification du copyright où la publication du site est interdit.
    Vous êtes l'auteur du document, pour la contribution du site, tout retrait peut être discuter, les adminstrateurs auront toujours le dernier mot, voir (Droit d'auteur).

    Vous aurez enfin votre propre repo, ou plutot un fork (Vous pouvez le supprimez à tout moment via les settings puis général, section "Danger Zone").

    2. Comment créer votre document

    Pour créer un document il faut créer un commit, un commit est une sauvegarde des modifications de votre repo.
    Pour modifier le fork il y a plusieurs façon.

    1. Créer votre codespace.
    2. (Recommender) Créer un fichier séparemment et le télécharger sur le repo. Cliquez sur un fichier puis "ADD file".

    Le format du document sera expliquer dans le guide de rédaction.

    Guide Rédaction

    Format du document

    1. Sur discord

    Le format est libre dans le contenu, tout dépend du texte que vous voulez créer, nous pouvons selon si ce n'est pas conforme au lien avec le lore, proposer des améliorations ou refuser.

    Sur le formulaire qui va s'ouvrir, vous pourrez voir le nom du document, cela correspond au nom du fichier qui va être créer "Document-XXX-XXX-XXX-etc.md", il faut utiliser des acronymes pour chaque XXX si le mot est long, la nomenclature doit correspondre à celle ci.

    • Document-NEX-"Contenu": Sujet
    • Document-NEX-XXX-"Contenu": Sous sujet
    • Document-NEX-XXX-XXX-"Contenu": Numéro du sujet
    • Document-NEX-XXX-XXX-XXX-"Contenu": Sous numéro du sujet
    • Document-NEX-XXX-XXX-XXX-XXX-"Contenu": Et encore...

    Le titre corresponds au nom du document quand il sera publier sur ce site, le nom du fichier n'apparaitra jamais.
    Le contenu correspond à ce que vas contenir votre document en lien avec le lore et ce que vous aimerez rajouter.

    2. Sur github

    Le format est libre dans le contenu, tout dépend du texte que vous voulez créer, nous pouvons selon si ce n'est pas conforme au lien avec le lore, proposer des améliorations ou refuser.

    Le nom de votre fichier devra contenir une nomenclature, elle correspondra à celle ci:

    • Document-NEX-"Contenu": Sujet
    • Document-NEX-XXX-"Contenu": Sous sujet
    • Document-NEX-XXX-XXX-"Contenu": Numéro du sujet
    • Document-NEX-XXX-XXX-XXX-"Contenu": Sous numéro du sujet
    • Document-NEX-XXX-XXX-XXX-XXX-"Contenu": Et encore...

    au début de votre document, vous devrez placer un frontmatter YAML pour votre titre
    à la fin votre document devra posséder la date de création (ou de mis à jour en plus de celle de création).

    La rédaction

    Pour créer le contenu pertinent de votre document faut prendre en compte cinq choses.

    1. Répondre aux 4 questions
    2. Orthographe soigner
    3. Avoir un lien au lore
    4. Ton respecter
    5. Le lien temporel respecter

    Les 4 questions

    Les questions peuvent être répondu sans le mettre dans le fichier. C'est à dire que l'auteur d'un document doit répondre aux 4 questions malgré qu'il ne soit pas marquer sur le fichier.
    Il peut bien évidemment répondre à ces questions dans son présent document.

    1. Quand se passe ce document ?
    2. Pourquoi ? dans quel but ?
    3. Pour qui ?
    4. Par qui ?
